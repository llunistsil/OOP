# Лабораторная работа №1: Вендинговый автомат

## Задание

Разработать консольное приложение, эмулирующее вендинговый автомат, позволяющее пользователю:

- Посмотреть список доступных товаров с их ценами и количеством
- Вставить монеты разных номиналов
- Выбрать товар и получить его, если внесённой суммы достаточно
- Получить сдачу (если нужно) и вернуть неиспользованные монеты при отмене операции
- Администраторский режим для пополнения ассортимента и сбора собранных средств

## Описание решения

### Архитектура приложения

Приложение построено на основе **объектно-ориентированного подхода** с разделением ответственности между классами.

#### Основные компоненты:

**1. Монеты (Coin hierarchy)**
- Абстрактный класс `Coin` с методом `GetValue()` для получения номинала
- Конкретные реализации: `CopperCoin` (1), `NickelCoin` (10), `PlatinumCoin` (100), `DiamondCoin` (1000)
- Интерфейс `IMonetaryValue` для объектов, имеющих денежную ценность
- Класс `CoinCollection` хранит коллекцию доступных монет, отсортированных по номиналу

**2. Товары (Product & Inventory)**
- Класс `Product` хранит информацию о товаре: название, цена, количество
- Класс `Inventory` управляет коллекцией товаров с методами добавления и поиска
- По умолчанию загружаются товары: cola (3, 25 шт.), candy (2, 15 шт.)

**3. Операторы (Operator pattern)**
- Абстрактный класс `Operator` - базовый класс для обработчиков команд
- `AdministratorOperator` - обработка команд администратора (добавление товаров)
- `CustomerOperator` - обработка команд покупателя (выбор товара, внесение монет, покупка)

**4. Вендинговый автомат (VendingMachine)**
- Центральный класс, координирующий работу всех компонентов
- Управляет переключением режимов (администратор/покупатель)
- Хранит накопленные средства в `_accumulatedFunds`
- Реализует логику расчета сдачи через `CalculateChange()`

### Режимы работы

**Переключение режимов:**
- Команда `switch` переключает между режимами admin/user
- Пустой ввод (Enter) продолжает работу в текущем режиме

**Режим администратора (admin):**
- При входе автоматически списывается вся накопленная сумма в автомате
- Команда `add` - добавить товар (новую позицию или пополнить существующую)
- Команда `quit` - выход из режима администратора
- Команда `help` - показать доступные команды

**Режим пользователя (user):**
1. Выбор товара из списка доступных
2. Внесение монет командами: `copper`, `nickel`, `platinum`, `diamond`
3. Команда `purchase` - завершение покупки
4. Автоматический расчет и выдача сдачи
5. Прозрачное переключение между режимами для эмуляции

### Выход из приложения

- Стандартный выход: `Ctrl+C`
- Обработчик `Console.CancelKeyPress` обеспечивает корректное завершение работы

## Используемые принципы ООП

- **Инкапсуляция**: данные классов скрыты, доступ через публичные методы
- **Наследование**: иерархия монет и операторов
- **Полиморфизм**: различные типы монет и операторов используются через базовые ссылки
- **Абстракция**: интерфейсы `IMonetaryValue` и абстрактные классы `Coin`, `Operator`
