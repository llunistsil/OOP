# Лабораторная работа №2: Система управления учебными курсами

## Задание

Создать систему для управления учебными курсами и преподавателями. Система должна:

- Позволять добавлять и удалять курсы, а также назначать преподавателей на эти курсы
- Включать возможность хранения и получения информации о студентах, записанных на каждый курс
- Поддерживать различные типы курсов: онлайн-курсы и офлайн-курсы с уникальными характеристиками
- Обеспечивать возможность получить все курсы, которые ведет конкретный преподаватель
- Ключевая бизнес-логика должна быть покрыта unit-тестами (xUnit)

## Описание решения

### Архитектура приложения

#### 1. Базовые сущности (абстрактные классы)

**Person (Человек)**
- Абстрактный базовый класс для всех людей в системе
- Свойства: `Id`, `FirstName`, `LastName`, `Email`
- От него наследуются:
  - `Learner` (Студент) - с дополнительным свойством `EnrollmentYear`
  - `Instructor` (Преподаватель) - с дополнительным свойством `Department`

**TrainingModule (Курс)**
- Абстрактный базовый класс для всех типов курсов
- Свойства: `Id`, `Name`, `Description`, `AssignedInstructor`, `EnrolledLearners`
- Методы: `SetInstructor()`, `AddLearner()`, `RemoveLearner()`, `GetTypeLabel()`
- От него наследуются:
  - `RemoteCourse` (Онлайн-курс) - с полями `PlatformUrl`, `IsSelfPaced`
  - `ClassroomCourse` (Офлайн-курс) - с полями `RoomNumber`, `Capacity`

#### 2. Сервисы для управления сущностями

**TrainingService (Сервис курсов)**
- Управляет коллекцией курсов в `Dictionary<string, TrainingModule>`
- Методы: `AddModule()`, `RemoveModule()`, `GetModule()`, `GetAllModules()`
- `GetModulesByInstructor()` - получение всех курсов преподавателя
- `AssignInstructorToModule()` - назначение преподавателя на курс

**InstructorService (Сервис преподавателей)**
- Управляет коллекцией преподавателей
- Методы: `AddInstructor()`, `GetInstructor()`, `RemoveInstructor()`, `GetAllInstructors()`

#### 3. Паттерн Строитель (Builder Pattern)

**Иерархия строителей:**
- Абстрактный класс `BaseBuilder<TBuilder, TObject>` - базовая реализация
- Интерфейс `ICreator<TObject>` - общий интерфейс для создателей
- `RemoteCourseBuilder` - построитель онлайн-курсов с методами:
  - `WithId()`, `WithName()`, `WithDescription()`
  - `WithPlatformUrl()`, `WithSelfPaced()`
- `ClassroomCourseBuilder` - построитель офлайн-курсов
- `StudentBuilder`, `TeacherBuilder` - построители для людей

**Особенности реализации:**
- Наследование от `Builder` для демонстрации паттерна
- Fluent Interface для цепочки вызовов
- Метод `Execute()` для выполнения операций над объектом

#### 4. Система управления университетом

**Интерфейсы (разделение ответственности):**
- `IEduManagementSystem` - главный интерфейс, объединяющий 4 маленьких
- `IInstructorManager` - управление преподавателями
- `ITrainingModuleManager` - управление курсами
- `IInstructorAssignment` - назначение преподавателей
- `ILearnerEnrollment` - запись студентов

**Класс `EduManagementSystem`:**
- Единая точка входа для клиентского кода
- Использует сервисы `TrainingService` и `InstructorService`
- Реализует все 4 интерфейса для масштабируемости
- Метод `HandleOperation()` для обработки исключений

### Принципы проектирования

**Интерфейсы:**
- Каждый интерфейс описывает отдельный способ взаимодействия
- Разбиение большого интерфейса на 4 маленьких для будущего масштабирования
- Следование принципу Interface Segregation (I из SOLID)

**Покрытие тестами:**
- `HumanTests.cs` - тесты для сущностей людей
- `CourseTests.cs` - тесты для курсов
- `CourseServiceTests.cs` - тесты сервиса курсов
- `TeacherServiceTests.cs` - тесты сервиса преподавателей
- `UniversityManagementSystemTests.cs` - интеграционные тесты системы

## Используемые паттерны и принципы

- **Builder** - пошаговое создание сложных объектов
- **Service Layer** - разделение бизнес-логики и представления
- **Interface Segregation** - разделение интерфейсов по ответственности
- **Dependency Injection** - внедрение зависимостей через конструктор
